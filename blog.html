1. What is the difference between map, foreach, filter and find?

Ans: Map, forEach, filter, and find are all methods available in JavaScript used to perform operations on arrays. However, each method has a specific purpose and functionality:

  1. Map : This method creates a new array by applying a given function to each element of the original array. The new array will have the same number of elements as the original array. It is often used to transform data in an array.
    Example:
    const arr = [1, 2, 3, 4, 5];
    const newArr = arr.map(num => num * 2);
    console.log(newArr);
    Output: [2, 4, 6, 8, 10]
  2. ForEach : This method calls a provided function once for each element in an array. It does not return a new array, but it can modify the original array.
    Example:
    const arr = [1, 2, 3, 4, 5];
    arr.forEach(num => console.log(num));
    Output:
    1
    2
    3
    4
    5
  3. Filter : This method creates a new array with all elements that pass the test implemented by the provided function. It returns a new array containing only the elements that meet the condition specified in the callback function.
    Example:
    const arr = [1, 2, 3, 4, 5];
    const filteredArr = arr.filter(num => num % 2 === 0);
    console.log(filteredArr);
    Output: [2, 4]
  4. Find : This method returns the value of the first element in the array that satisfies the provided testing function. If no value satisfies the function, undefined is returned.
    Example:
    const arr = [1, 2, 3, 4, 5];
    const foundNum = arr.find(num => num > 3);
    console.log(foundNum);
    Output: 4

In summary, map and filter both create new arrays, while forEach and find do not. forEach is used to loop over the elements of an array and perform an action on each element, while find is used to find the first element in an array that satisfies a condition.

2. Difference between Local Storage, Session Storage and Cookies?

Ans: Local Storage, Session Storage, and Cookies are all used to store data in the user's browser, but there are some key differences between them:

  1. LocalStorage :
    LocalStorage is a mechanism for storing key-value pairs in the browser. The data stored in LocalStorage remains available even after the browser is closed. LocalStorage is limited to a maximum of 5-10 MB of data per domain depending on the browser. LocalStorage can be used to store information such as user preferences, settings, and other data that needs to persist across sessions.
  2. SessionStorage :
    SessionStorage is similar to LocalStorage, but the data stored in SessionStorage is available only for the duration of the browser session. When the browser is closed, the data stored in SessionStorage is deleted. SessionStorage is also limited to a maximum of 5-10 MB of data per domain depending on the browser. SessionStorage is commonly used to store information such as shopping cart items, form data, and other temporary data.
  3. Cookies :
    Cookies are small pieces of data that are sent from a website to a user's browser and stored on the user's computer. Cookies can be set to expire after a specific time or when the browser is closed. Cookies are limited to a maximum size of 4 KB per domain. Cookies are commonly used to store user preferences, login information, and other data that needs to persist across sessions.

In summary, LocalStorage and SessionStorage are similar in that they allow you to store data on the client-side, but SessionStorage is cleared when the browser is closed, while LocalStorage is not. Cookies are also used to store data on the client-side, but they are limited in size and can be set to expire after a specific time.

3. What is Event loop and how it works??

Ans: The Event Loop is a critical concept in JavaScript and other programming languages that utilize an event-driven or asynchronous programming model. The event loop is a mechanism that allows JavaScript to handle and execute multiple tasks simultaneously without blocking the main execution thread.
In JavaScript, the event loop is responsible for managing the execution of asynchronous tasks such as timers, I/O operations, and event handlers. When an asynchronous task is started, it is added to the task queue. The event loop continually checks the task queue and picks up the next task when the current task has completed. This allows for non-blocking asynchronous code execution, which is essential for building responsive and efficient applications.
The event loop works as follows:

  • When an asynchronous task is started, it is added to the task queue.
  • The event loop continually checks the task queue for any pending tasks.
  • When the current task has completed, the event loop picks up the next task from the queue and executes it.
  • This process repeats until there are no more tasks in the queue.
It's important to note that the event loop is a single-threaded process, which means that only one task can be executed at a time. However, because the event loop allows for non-blocking execution, other tasks can be queued and processed while the current task is waiting for I/O or other operations to complete.
